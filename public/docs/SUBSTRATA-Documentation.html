1. Overview

SUBSTRATA is a full-stack prototyping command center. AI image generation, parametric 3D modeling, laser G-code pipelines, label/sticker design, real-time material analysis, wiring diagrams, and firmware scaffolding all live in one dark-mode glassmorphism cockpit powered by Google Gemini 3.1.

Three engineering modes — 3D Prototype, Laser Studio, and Labels — cover the full path from idea to finished product. A persistent AI Design Advisor co-pilots every mode, with access to 30+ real components, 12 project archetypes, and DFM best practices.

Edge Silhouette Outline

The silhouette feature generates a vector cut path from any design image so stickers can be laser-cut to their exact shape:

  1. Binary Mask: Uses alpha channel or luminance thresholding to separate content from background
  2. Boundary Detection: Identifies edge pixels (content adjacent to background)
  3. Angular Sort: Orders edge points by angle from centroid for continuous path
  4. Path Simplification: Reduces point count (every Nth point) for cleaner SVG
  5. Offset Cut Line: Generates 2px outward offset from silhouette for kerf compensation
  6. SVG Output: Red line = laser cut path, Blue line = design boundary reference

Architecture Mode

Backed by buildingCodeRules.ts, electricalPlan.ts, and layerSystem.ts:

  • Building code validation — checkBuilding(descriptor) evaluates doors, stairs, ramps, and corridors against IBC and ADA citations. Emits a BuildingCodeReport with rule, severity, message, suggested remediation.
  • IBC-1010.1.1 / ADA-404.2.3 — Door clear width 32–36" (815–915 mm), 80" (2032 mm) headroom.
  • IBC-1011.5.2 — Stair risers 4–7" (102–178 mm), treads ≥ 11" (279 mm), width ≥ 36".
  • ADA-403.3 — Ramp slope ≤ 1:12 (8.33 %), landing minimums, handrail rules.
  • Electrical plan & panel schedule — buildPanelSchedule(), validateElectricalPlan() (NEC clearance, GFCI/AFCI, grounding), renderElectricalSvg().
  • Layer system (AIA / ISO-16739) — 40+ preset layers managed by LayerManager: A-WALL, A-DOOR, A-WIND, A-FLOR, A-ROOF, A-CEIL, A-STAIR, A-FURN, A-EQPM, A-COLS, S-COLS, S-BEAM, S-FNDN, S-SLAB, M-HVAC-*, P-PIPE-*, E-LITE, E-POWR, E-COMM, E-PANL, C-PROP, C-TOPO, C-PKNG, C-WALK, L-PLNT, A-ANNO-* and more. Toggle visibility, lock, assign parts to layers, export DXF-style maps.

Hacker Mode

Hacker mode wires Gemini straight into a canonical schematic IR (circuitGraph.ts) and emits real, importable KiCad files (kicadEmit.ts):

  • generatePCBSchematic(prompt) — Gemini Pro is constrained to output a typed Schematic (sheets → components → pins → nets) instead of free-form text.
  • validateSchematic() — Unique reference designators, declared pins, valid net connections, canonical power-net naming (VCC, GND, +3V3).
  • KiCad 8 / 9 emitter — emitProjectFiles(schematic) writes a real .kicad_sch S-expression (symbols, properties, UUIDs, pin positions, net labels) plus a project README. Drop the folder into KiCad and it opens.
  • Component pin library — ESP32 DevKit, Arduino Nano, RPi Pico, A4988 / TMC2209 drivers, OLED, MPU6050, HC-SR04, WS2812B, and more.

Reference Library & Smart Blocks

Open-Access Scraper

The Library modal is fed by a pluggable ScraperAdapter framework in src/lib/scraper. Two adapters ship by default — both license-clean by construction.

SourceLicenseSpecialty
Smithsonian Open AccessCC0 1.03D models, photographs, museum artifacts
Library of Congress HABS / HAER / HALSPublic DomainHistoric American building plans, measured drawings
  • License-aware search — SpdxLicense enum (CC0-1.0, CC-BY-4.0, CC-BY-SA-4.0, PD, MIT, Apache-2.0). isPermissive() defaults search to safe-to-remix sources only.
  • Filterable hits — Filter by kind (3d_model, blueprint, floor_plan, photograph) and format (stl, glb, gltf, obj, step, svg, dxf, pdf, ifc).
  • Polite fetching — Per-host rate limiting (throttle.ts) and SHA-256 deduplication (hash.ts).
  • Attribution capture — Every FetchedAsset carries licenseProof and attributionString.
  • Direct import — "Use" button drops a hit into the active project as a base mesh, blueprint reference, or texture.

Smart Blocks

smartBlocks.ts audits generated OpenSCAD code and refactors repeated geometry into reusable modules.

  • detectRepeatedGeometry() — Signature-matches primitives by type + normalized aspect ratio.
  • scoreOpenSCADCandidates() — Ranks candidates by potential savings (instance count × size).
  • convertBlocksInOpenSCAD() — Rewrites inline primitives as named module name() { ... } definitions and translate() name(); call sites.
  • auditOpenSCADBlocks() — Returns a SmartBlocksReport with total primitives, candidate blocks, existing modules, projected line savings, and conversion suggestions.

3D File Validation & DFM

meshValidator.ts gives every imported STL/GLB/OBJ a printability scorecard:

  • Closed-mesh / non-manifold-edge / self-intersection detection.
  • Wall-thickness check vs. printer minimums (FDM 1.2 mm · SLA 0.5 mm).
  • Overhang detection (> 45° unsupported), bridge-capacity warnings (> 10 mm spans).
  • Polygon count, estimated print time, weight (g), and cost (USD).
  • validateOpenSCADForPrinting() applies the same DFM ruleset to generated code before fabrication.
